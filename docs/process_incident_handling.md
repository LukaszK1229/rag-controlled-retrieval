# Incident Handling Process

## Purpose
Define how operational incidents are reported, handled, and escalated.

## What is an Incident
An incident is any event that disrupts normal operations or impacts customers.

## Reporting
All incidents must be reported via the internal ticket system.

## Severity Levels
- SEV-1: Critical outage affecting customers
- SEV-2: Major issue with workaround
- SEV-3: Minor issue, no immediate impact

## Escalation Rules
- SEV-1 → Ops Lead + CTO immediately
- SEV-2 → Ops Lead within 30 minutes
- SEV-3 → handled by on-duty Ops

## Resolution
Incident is closed only after:
- root cause is documented
- fix is deployed
- Ops Lead approves closure
